View encapsulation

To prevent overriding global styles or other components design Angular encapsulates the styling of each component. There are 3 encapsulation modes that can be selected.

ViewEncapsulation.ShadowDom
  • Uses the browser build-in shadow-dom
  • Encloses the component into a ShadowRoot
  • Components content projected into that ShadowRoot, so the applied styling does not affect other components
ViewEncapsulation.Emulated
  • Emulates the shadow dom api
  • Modifies each css selector in that component and append the styling on it
  • Because of that the styling would also not affect other components
 ViewEncapsulation.None
  • Does not apply any view encapsulation
  • Styling affects the global app like it would be defined in a global stylesheet

Generated CSS in emulated mode

In emulated mode Angular pre-processes every element in a component and simulates the behaviour of the shadow dom. The root element gets an attribute called _nghost and each child-element gets the attribute _ngcontent. The styling for those pre-processed elements will be expanded with a post-processed styling-selectors in their stylesheet: selector[_nghost] and selector[_ngcontent].

Mixing encapsulation modes

It is not recommended to mix these modes wildly together. If needed be aware of the consequences!
