Styles in the component metadata
Components can have multiple style declarations within the respective decorator. The value from this attribute is a string array. Each entry defines exclusive CSS for that component. styles: ['h1 ❴ font-weight: normal; ❵']
Style-Files in the component metadata
If CSS for a specific component is defined in an external file, it can be referenced within the component decorator with the styleUrls-Attribute. It contains a string array with (possible) multiple CSS-Files. styleUrls: ['./hero-app.component.css']
Template inline styles
Styles could also be applied within the styles tag in the components template.
Template link tags
Link-Tags can be used within the components template to load external CSS-Files. This external files should be considered during the build-stage.
CSS @imports
CSS-Files can be imported into other css files with the @import statement and a relative path to that file.
External and global style files
External CSS-Files have to be included while building the angular application. To achieve this, there is a section "styles" in the angular.json file, where the styles.css is already entered by default. Other files can be added here.
Non-CSS style files
External CSS-Files have to be included while building the angular application. To achieve this, there is a section "styles" in the angular.json file, where the styles.css is already entered by default. Other files can be added here.
