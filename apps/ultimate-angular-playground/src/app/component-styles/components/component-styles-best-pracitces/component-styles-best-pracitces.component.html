The styles belong to the private implementation of the component. It is therefore not advisable to change internal structures via CSS. Angular prevents this intervention by default via view encapsulation. Global styles are nevertheless applied across all components.

CSS Custom Properties (recommended)
Using CSS variables, components allow embedding elements to override specific styles. The component itself defines variables such as the color of the heading. The parent component then overrides this variable with a different color value if the default case is not sufficient.
Declare global CSS with @mixins
Angular denies any overrides of internal CSS structures through view encapsulation. By defining @mixins in the global CSS file, developers can define different states of the component such as a "Dark" or "Light" theme. The respective components can then define themselves how they should look in the respective @mixins.
Using ::part as pseudo-element
The ::part element describes a normal HTML element with a part attribute. This can contain either one or more labels as value. These are also present in the shadow DOM. For this reason, a parent component can redesign parts of the child component using the ::part selector.
Provide a TypeScript API (not recommended)
Component developers can define a TypeScript API that allows other developers to dynamically modify the styles of a component. However, this is not recommended because the additional (regular) execution of the JavaScript blocks can lead to performance issues.
